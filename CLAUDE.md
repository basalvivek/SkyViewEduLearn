## Content Hierarchy (tree structure)

```
Category  (e.g. Year 10)
  └── Subject  (e.g. English Language)
        └── Topic  (e.g. Non-Fiction & Creative Writing)
              └── Question
```

Questions support 7 types: `MCQ_SINGLE`, `MCQ_MULTIPLE`, `TRUE_FALSE`, `SHORT_ANSWER`, `ESSAY`, `CODE`, `IMAGE_BASED`.

---

## Approval Workflow Rules

1. Admin creates any content → auto status `APPROVED`.
2. Teacher creates any content → status starts as `DRAFT`.
3. Teacher clicks submit → status becomes `PENDING`.
4. Admin approves → `APPROVED`. Admin rejects (with reason) → `REJECTED`.
5. Teacher edits any APPROVED or REJECTED item → re-triggers `PENDING` on submit.
6. Admin-owned nodes are **read-only** for teachers in the UI (teacher can add children inside but cannot edit or delete the node itself).
7. `rejection_reason` field must be stored and returned so teacher can see why it was rejected.

---

## Database Rules

- Use schema from `sql/02-database-schema.sql` exactly — run it, do NOT use `ddl-auto=create`.
- All PKs are `UUID` generated by PostgreSQL `gen_random_uuid()`.
- Every content table has: `status`, `rejection_reason`, `created_by`, `updated_by`, `approved_by`, `approved_at`, `is_deleted`, `deleted_at`, `created_at`, `updated_at`.
- Soft delete: set `is_deleted=true`, `deleted_at=NOW()` — never hard delete.
- Use `@Where(clause = "is_deleted = false")` and `@SQLDelete` on every entity.
- Approval changes are logged in `approval_log` table.

## API Design

Base: `http://localhost:8080/api/v1`

All responses use:
```json
{ "status": "success|error", "message": "...", "data": {}, "pagination": null }
```

### Auth
- `POST /api/v1/auth/login` — public — returns `{ token, role, name }`
- `GET  /api/v1/auth/me` — authenticated

### Content (all four levels follow same pattern)
- `GET    /api/v1/categories` — any (admin sees all statuses, others see APPROVED only)
- `POST   /api/v1/categories` — ADMIN or TEACHER
- `GET    /api/v1/categories/{id}` — any
- `PUT    /api/v1/categories/{id}` — owner or ADMIN
- `DELETE /api/v1/categories/{id}` — ADMIN only (soft delete)
- `PUT    /api/v1/categories/{id}/approve` — ADMIN only
- `PUT    /api/v1/categories/{id}/reject`  — ADMIN only, body: `{ "note": "reason" }`
- Subjects scoped under: `/api/v1/categories/{cid}/subjects`
- Topics scoped under:   `/api/v1/subjects/{sid}/topics`
- Questions scoped under:`/api/v1/topics/{tid}/questions`

### Tree
- `GET /api/v1/categories/{id}/tree` — returns full nested hierarchy

### Admin queue
- `GET /api/v1/approvals/pending?type=TOPIC&page=0&size=20` — ADMIN only

### Teacher view
- `GET /api/v1/my/submissions?status=PENDING` — returns teacher's own content by status

### Phase 1 — Org & Notifications
- `GET  /api/v1/org/settings` — get org settings (authenticated)
- `PUT  /api/v1/org/settings/branding|legal|address|contacts|academic` — ADMIN only
- `POST /api/v1/org/settings/logo` — upload logo, ADMIN only
- `GET  /api/v1/notifications` — current user's notifications
- `PUT  /api/v1/notifications/read-all` — mark all read

### Phase 2 — Teacher Workspace
- `GET  /api/v1/my/profile` — own profile
- `PUT  /api/v1/my/profile` — update own profile
- `POST /api/v1/my/profile/avatar` — upload avatar
- `PUT  /api/v1/my/profile/password` — change password (requires current password)
- `GET  /api/v1/my/submissions` — paginated with filters: status, type, date, search
- `GET  /api/v1/my/submissions/summary` — counts by status
- `PUT  /api/v1/{categories|subjects|topics|questions}/{id}/submit` — DRAFT/REJECTED → PENDING

### Phase 3 — Assessment Engine
- `GET/POST /api/v1/exams` — list / create exam
- `GET/PUT/DELETE /api/v1/exams/{id}` — manage exam
- `PUT /api/v1/exams/{id}/questions` — set ordered question list
- `PUT /api/v1/exams/{id}/submit|approve|reject` — approval workflow
- `GET/POST /api/v1/classes` — class management (Admin)
- `GET/POST /api/v1/schedules` — schedule an exam for a class
- `GET/POST /api/v1/students` — student accounts (Admin)
- `POST /api/v1/students/bulk-import` — CSV import (Admin)
- `POST /api/v1/students/{id}/classes` — assign student to class
- `GET  /api/v1/student/exams` — student's available exams
- `POST /api/v1/student/attempts` — start exam attempt
- `PUT  /api/v1/student/attempts/{id}/answers` — auto-save answers
- `POST /api/v1/student/attempts/{id}/submit` — final submit (triggers auto-marking)
- `GET  /api/v1/student/attempts/{id}/result` — view result
- `GET  /api/v1/marking/queue` — manual marking queue (Teacher/Admin)
- `PUT  /api/v1/marking/attempts/{aid}/answers/{qid}` — award marks + feedback
- `POST /api/v1/marking/attempts/{id}/finalise` — complete marking, compute final score

### Phase 4 — Reporting & Analytics
- `GET /api/v1/student/results` — student's own attempt list
- `GET /api/v1/student/results/{attemptId}` — full result with topic breakdown
- `GET /api/v1/results/schedule/{scheduleId}` — all student results for one schedule
- `GET /api/v1/results/schedule/{scheduleId}/export` — CSV export
- `GET /api/v1/reports/class/{classId}/summary|topics|students|distribution` — class reports
- `GET /api/v1/reports/teachers` — teacher activity (Admin only)
- `GET /api/v1/analytics/questions` — question performance table
- `PUT /api/v1/analytics/questions/{id}/flag|unflag` — flag for review

### Phase 5 — Communications
- `GET /api/v1/notifications` — current user's notifications (paginated)
- `GET /api/v1/notifications/unread-count` — badge count
- `PUT /api/v1/notifications/{id}/read` — mark one read
- `PUT /api/v1/notifications/read-all` — mark all read
- `GET/POST /api/v1/announcements` — list / create announcements
- `POST /api/v1/announcements/{id}/dismiss` — dismiss pinned banner

### Phase 6 — Polish & Operations
- `GET /api/v1/audit-log` — paginated audit log (Admin)
- `GET /api/v1/audit-log/export` — CSV export
- `POST /api/v1/import/validate` — validate CSV/Excel question import
- `POST /api/v1/import/commit` — commit validated import
- `GET /api/v1/settings` — all platform settings grouped by category
- `PUT /api/v1/settings/batch` — update multiple settings
- `GET /api/v1/export/questions` — question bank as Excel
- `GET /api/v1/export/exam/{id}/paper` — exam paper as PDF
- `GET /api/v1/export/results/schedule/{id}` — class results as Excel

## Security Rules

- JWT stored in browser `sessionStorage` as `edu_token`.
- Every API call sends `Authorization: Bearer <token>`.
- Public routes (no token needed): `POST /auth/login`, `GET /categories/**` (approved only), all static files.
- ADMIN-only routes: `DELETE /**`, `/approvals/**`.
- Role check: use `@PreAuthorize("hasRole('ADMIN')")` or check in service layer.
- CORS: allow only `http://localhost:8080` and `http://127.0.0.1:8080`.

## Error Handling

`GlobalExceptionHandler` must handle:
- `ResourceNotFoundException` → 404
- `ForbiddenException` → 403
- `MethodArgumentNotValidException` → 400 with field errors
- `DataIntegrityViolationException` → 409
- All others → 500

All errors use the standard `ApiResponse` envelope.
